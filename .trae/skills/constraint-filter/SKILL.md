# 硬约束过滤 — 核心过滤模块

## 核心使命

> 用最严格的刚性条件，为考生圈出一个"100% 可以合法填报"的基本盘；任何不能确定的，宁可排除在外或明示警告，也不混进基本盘假装可以。

详尽的设计背景、哲学与错误处理规约见：`../../../prds/硬约束过滤-SKILL需求规格说明书.txt`。

## 何时触发

任一条件满足即应调用：

- 用户出现 "志愿过滤 / 筛选 / 硬约束 / 能不能报 / 选科要求 / 色盲能否报化学 / 政审 / 体检限制" 等意图；
- 工作流中已完成考生画像收集，需要给下游「专业评估」模块喂入基本盘；
- 用户上传考生信息 JSON 并要求"输出可选院校专业"。

不触发：评估专业好坏、推荐院校、排冲稳保顺序（交给下游相关模块）。

## 使用方法

### 方式一：Python 模块调用（推荐）

```python
from pathlib import Path
import sys

sys.path.insert(0, str(Path("<此 skill 绝对路径>")))
from constraint_filter import run

```

### 考生画像 MD 格式（AI 必须按此格式创建文件）

收集完考生信息后，**必须**将考生画像保存为 MD 文件，路径固定为：
`结果/1初步筛选_考生画像_<省份>.md`（如 `结果/1初步筛选_考生画像_山东.md`）

文件内容格式如下（严格按此格式，字段名不可变）：

```markdown
# 考生画像

- 省份：山东
- 总分：566
- 省内位次：54147
- 高考模式：综合改革
- 选科：物理、化学、生物
- 性别：男
- 考试年份：2024
- 政审清白：是

## 体检信息

- 色觉：正常
- 视力左：4.8
- 视力右：4.8
- 身高(cm)：175
- 体重(kg)：65
- 听力正常：是
```

创建完成后，将此 MD 文件路径传给 CLI：

```bash
python main.py 结果/1初步筛选_考生画像_山东.md
```

### 方式二：CLI

```bash
# 推荐：自动保存到项目根目录的 结果/ 文件夹
python main.py 结果/1初步筛选_考生画像_山东.md
```

退出码：`0` 表示 success，`2` 表示任何 error 响应。

### 下游模块消费方式

下游模块（如 major-evaluator）直接读取 `结果/1初步筛选_硬约束过滤结果_<省份>.md`（如 `结果/1初步筛选_硬约束过滤结果_山东.md`）。
MD 文件包含完整的冲/稳/保分档表格和考生画像，可直接作为上下文传入。

### 输入结构

字段定义见 `schemas/input_schema.json`。必填：`province, total_score, provincial_rank, subject_category, selected_subjects, gender`。缺失任一字段或字段越界 → 抛出 `INPUT_MISSING_REQUIRED_FIELD` / `INPUT_INVALID_FIELD_VALUE` 并停止。

最小示例：

```json
{
  "province": "河南",
  "total_score": 580,
  "provincial_rank": 45000,
  "subject_category": "物理类",
  "selected_subjects": ["物理", "化学", "生物"],
  "gender": "男",
  "physical_exam": {"color_blindness": "正常", "vision_left": 4.8, "vision_right": 4.8,
                     "height_cm": 175, "weight_kg": 65, "hearing_normal": true,
                     "other_conditions": []},
  "political_review_clean": true,
  "exam_year": 2026
}
```

### 输出结构

见 `schemas/output_schema.json`。两种状态二选一：

- `status=success`：含 `eligible_pool`、`excluded_summary`、`all_majors_detail`、`warnings`、`stats` 等字段。
- `status=error`：含 `error_code`、`error_message`、`user_action_required`、`error_detail`。

**凡异常一律走 error 分支，绝不降级为 success。**

#### `all_majors_detail` 字段说明

`all_majors_detail` 是招生计划中**所有专业**的逐条过滤结论（不论通过与否），每条记录包含：

| 字段 | 含义 |
|------|------|
| `school_name` / `major_name` | 院校和专业名称 |
| `status` | `"eligible"`（通过）或 `"excluded"`（排除） |
| `exclude_dimension` | 排除发生在哪一层（选科/分数位次/体检/政审） |
| `exclude_reason` | 具体排除原因（人可读字符串） |
| `rank_tier` | 冲/稳/保/unknown（若已计算） |
| `rank_ratio` | 位次比值（若已计算） |
| `yearly_admission_data` | 历史录取位次数据（若已查找） |

这个字段是"没有瞎说"的凭证——AI 在展示结果时必须基于此字段逐校逐专业说明。

## AI 展示结果的规范

### 场景一：全量过滤输出（用户请求筛选所有可选项）

必须基于 `all_majors_detail` 逐校逐专业展示，不得用"100+"、"等多所"等模糊表述。

**推荐输出格式**（按学校分组，每专业一行）：

```
⚠️ 数据说明：以下筛选基于【2024年】{省份}招生录取数据，非{考生年份}官方数据。

## 【冲档】以下院校专业组 — 该考生可冲

| 学校 | 专业组 | 专业 | 位次比 | 近年最低录取位次 | 排除原因 |
|------|--------|------|--------|-----------------|----------|
| 山东大学 | 理工01组 | 计算机科学与技术 | 0.91（冲） | 38,200（2024） | — |

## 【稳档】以下院校专业组 — 大概率可录取

（同上表格格式）

## 【保档】以下院校专业组 — 极大概率录取

（同上表格格式）

## 被排除的选项（共 N 个专业）

| 学校 | 专业 | 排除原因 |
|------|------|----------|
| 北京大学 | 临床医学 | 分数位次维度：rank_ratio≈0.45，远超冲档上限 |
| 山东大学 | 化学工程 | 选科维度：要求"物理+化学"，考生选科不含化学 |
```

**铁律**：
- 每个专业必须出现在输出中（either 冲/稳/保表，or 排除表）
- 排除原因必须来自 `exclude_reason` 字段，不得自行发明
- 历史录取位次必须标注数据年份（如"38,200（2024年）"）
- 不得出现"需进一步筛选"、"等多所院校"、"前N个结果"

### 场景二：单校单专业查询（用户问"我能上X学校的Y专业吗？"）

当用户提问特定学校+特定专业时，从 `all_majors_detail` 中找到对应记录，输出以下结构：

```
## {学校名} · {专业名} — 资格分析

**结论**：✅ 可以填报（稳档）/ ⚠️ 可以尝试（冲档）/ ❌ 不建议填报（超出范围）/ 🚫 无资格填报（硬约束不符）

### 五层过滤结果

| 过滤维度 | 结论 | 依据 |
|----------|------|------|
| 省份招生计划 | ✅ 通过 | 该校在{省份}有招生名额 |
| 分数位次 | ✅ 稳档（rank_ratio=1.12） | 近年最低录取位次 38,200（2024年），考生位次 34,000 |
| 选科要求 | ✅ 通过 | 要求"物理+化学"，考生已选物理+化学+生物 |
| 体检限制 | ✅ 通过 | 该专业无特殊体检限制 |
| 政审 | ✅ 通过 | 非公安/军校类院校 |

### 历史录取位次参考

| 年份 | 最低录取位次 | 最低录取分数 |
|------|------------|------------|
| 2024 | 38,200 | 572分 |
| 2023 | 36,800 | 568分 |
| 2022 | 40,100 | 561分 |

⚠️ 以上为历史数据，仅供参考，实际填报以当年官方数据为准。

### 综合建议

rank_ratio = 考生位次(34,000) ÷ 近年加权平均最低位次(38,200) ≈ 0.89，属于冲档范围。
录取概率约30-50%，建议作为1-2个冲档志愿之一，需搭配足够的稳档和保底志愿。
```

**铁律**：
- 如果 `all_majors_detail` 中没有该专业（不在招生计划内），明确告知"该校在{省份}无招生计划，无法填报"
- 五层过滤结果必须逐层列出，不得跳过任何一层
- 历史位次表必须显示原始数据和年份，不得仅说"历史最低位次约38000"
- rank_ratio 的计算过程必须透明展示（让用户能核对）

## 五层过滤流程（固定顺序）

1. 省份与招生计划过滤（加载 `data/admission_plans/{省}_{年}.json`）
2. 分数位次过滤（rank_ratio 加权平均 + 冲/稳/保分档）
3. 选科/文理准入（逐专业组解析 `subject_requirement_raw` 为逻辑表达式）
4. 体检限制（以 `data/physical_restrictions.json` 为唯一依据，禁止推断）
5. 政审与特殊限制（public security / military / 定向等）

前一层排除的专业立即退出，不再进入下游层。每一次排除都记录在 `excluded_summary`，用户可逐条核对。

## 设计不变式（违反则判代码不合规）

- **数据没有就说没有**——任何缺失字段都不得用"同校均值""估算位次"等方式补。
- **解析失败就报错**——选科要求字符串不符合 `subject_requirement_keywords.json` 规则时，抛 `RUNTIME_SUBJECT_REQUIREMENT_PARSE_FAILED`。
- **版本必须匹配**——招生计划文件 `valid_for_year` ≠ 考生年份时抛 `DATA_VERSION_MISMATCH`。
- **体检维度以表为准**——映射表未覆盖的专业标 `physical_check: not_verified`，绝不自行"推测"它是否限色盲。
