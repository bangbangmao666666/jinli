# special-advisor — 特殊适配与终审模块: 特殊适配

## 核心使命

> 对 填报策略模块（strategy-planner 生成的志愿方案初稿执行最终审核和特殊群体适配，用特殊群体规则做精调，生成一份安全、完整、透明的最终志愿填报报告；任何不确定的风险宁可标注出来让用户自行确认，也绝不隐瞒或替用户做主。

详尽的设计背景、哲学与错误处理规约见：`../../../prds/special-advisor-SKILL需求规格说明书.txt`。

## 何时触发

任一条件满足即应调用：

- 上游 填报策略模块 输出志愿方案初稿完成，需要最终审核；
- 用户出现 "女生报什么专业好"、"女孩子适合学什么"、"女生报XX专业有歧视吗" 等意图；
- 用户出现 "刚过本科线怎么办"、"分数很低怎么报"、"专科和本科怎么选" 等意图；
- 用户出现 "想考公报什么专业"、"哪些专业好考编"、"选调生需要什么学校" 等意图；
- 用户出现 "要不要复读"、"复读一年值得吗" 等意图；
- 用户要求 "帮我看看这个方案安全吗"、"这个志愿顺序有问题吗"。

## 一、中间数据获取规则（最高优先级）

**所有中间数据必须从 `结果/` 目录读取，禁止从项目根目录或其他位置读取同类文件。**

| 数据类型 | 读取路径 |
|---------|---------|
| 考生画像 | `结果/1初步筛选_考生画像_<省份>.md` |
| 填报策略方案 | `结果/4填报策略_填报策略方案_<省份>.md` |
| 专业评估报告 | `结果/2选专业_专业评估报告_<省份>.md` |
| 院校匹配结果 | `结果/3选学校_院校匹配结果_{省份}.md` |

**操作顺序**：依次读取上游 模块 的输出文件。如 `结果/` 目录下无对应文件，停止并提示用户按顺序运行。

## 使用方法

### 方式一：Python 模块调用（推荐）

```python
from pathlib import Path
import sys

sys.path.insert(0, str(Path("<此 skill 绝对路径>"))
from special_advisor import run

result = run(upstream_output, applicant_profile)  # 返回 PRD §3 定义的结构化 dict
```

### 方式二：CLI

```bash
python main.py input.json --pretty
# 或从 stdin
cat input.json | python main.py -
```

退出码：`0` 表示 success，`2` 表示任何 error 响应。

### 输出结构

见 PRD §3.1"最终志愿填报报告结构"。两种状态二选一：

- `status=success`：含完整的最终报告结构。
- `status=error`：含 `error_code`、`error_message`、`user_action_required`、`error_detail`。

**结果写入**：每次运行成功后，必须将最终终审意见保存为：`结果/5精调终审_方案终审意见_<省份>.md`。

**凡异常一律走 error 分支，绝不降级为 success。**

## 特殊适配规则集

### 规则集一：中低分段女生

- 移除存在就业性别歧视的专业；
- 提升师范类、护理类、财务类等女生友好专业优先级；
- 专业组干净度加严审核。

### 规则集二：数理基础薄弱

- 对高数理要求专业加高危警告；
- 推荐应用型替代方向。

### 规则集三：体制内目标

- 专业向考公目录倾斜；
- 优先推荐有选调生资格的院校。

### 规则集四：低分段考生

- 评估"保本科 vs 选好专科"；
- 推荐优质专科方向。

### 规则集五：复读可能性评估

- 客观列出复读利弊；
- 不主动建议复读。

## 最终审核检查清单

### 安全性检查

- 基本盘范围
- 隐藏门槛
- 兜底安全
- 专业组干净度
- 学费承受力

### 适配性检查

- 体制内外
- 地域
- 经济
- 性别
- 数理基础

### 完整性检查

- 冲稳保比例
- 备选方案
- 待确认事项
- 风险标注

## 设计不变式（违反则判代码不合规）

- **通用方案是底座，特殊适配是精调——不推翻上游 模块 结论，只微调；
- **有风险就标注**——宁可误标也不能漏标；
- **不替用户做主**——人生级决定必须由考生/家长自己做；
- **有据可查**——每个调整、每条警告都能回溯到明确规则；
- **极端透明**——所有不确定的风险都明确告知用户。
