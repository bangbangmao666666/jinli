# school-matcher — 院校匹配模块：院校匹配Skill

## 核心使命

> **在用户选定 3-5 个意向专业后**，用可回溯的数据和规则，对这些专业在基本盘中的院校组合进行八维深度评估，输出「专业+院校」匹配卡；任何维度缺乏数据支撑的，宁可标注「数据不足」，也不编造评价。


详尽的设计背景、哲学与错误处理规约见：`../../../prds/school-matcher-SKILL需求规格说明书.txt`。

---

## 精准评估工作流（架构核心，必须理解）

为了防止山东、河南等大省的基本盘导致输出文档过长（数万字）且失去重点，本 Skill 采用**“聚焦匹配”**策略：

```
第一阶段（Python 脚本）：结构性过滤
  输入：硬约束过滤模块 MD + 专业评估模块 MD + 用户选定的 3-5 个专业名称
  计算：仅提取选定专业的院校组合，计算 rank_ratio、档位分类、数据库线索标记
  输出：结果/3选学校_院校匹配结果_{省份}_{分数}.md（聚焦候选池，通常 30-60 条）
  职责边界：通过 selected_majors 字段实现精准减负，确保后续 AI 评估有重点

第二阶段（AI + WebSearch）：八维深度评估
  输入：第一阶段生成的聚焦候选池（读取文件末尾的「AI 评估待办」清单）
  动作：对清单中的院校-专业组合执行 WebSearch，按八维框架逐一评估
  输出：完整的推荐卡（每卡八维表+综合判断），作为 strategy-planner 的候选池
  职责边界：定性结论必须来自 WebSearch，输出卡片应覆盖用户选定的所有 3-5 个专业方向
```

**执行规则**：
1. **优先读取**用户在对话中明确提到的专业（如“我想看计算机和自动化”）。
2. **强制约束**：若用户未指定，必须先展示 专业评估模块 报告并请求用户选定 3-5 个方向，禁止直接运行全量匹配。
3. **输出要求**：对选定专业下的重点院校（冲5/稳15/保20原则）必须输出完整八维评估表。

## 执行前必读：四条硬性约束

> **这四条约束比本文件其他所有内容都优先。**

**约束 1：school-database.md 是评估框架，不是结论来源**

school-database.md 的价值在于它蒸馏了张雪峰的选校方法论，告诉你**该看哪些维度、问哪些问题**。但它收录的院校数量有限，且信息存在时效性问题，不能直接作为输出依据。

正确的使用方式：
- **评估框架**：八个维度（行业血统、性价比、名实反差、深造通道、就业资源、学科认可、办学性质、城市环境）来自该文件的方法论，适用于所有院校
- **搜索线索**：数据库中已有描述（如"长春理工是光学领域明珠"）作为 WebSearch 的初始假设和关键词，而非最终结论
- **所有输出结论**：无论该校是否在数据库中，都必须通过 WebSearch 获取有时效性的信源支撑，标注「🌐 [具体信源名称，年份]」
- 禁止将 school-database.md 中的描述原文直接搬入输出；禁止用「综合评估」等模糊措辞掩盖无据推断；禁止用模型训练知识替代搜索

**约束 2：rank_ratio 异常值必须处理**
- rank_ratio 计算方式：`考生位次 ÷ 历史最低录取位次加权平均`
- rank_ratio < 0.3：数据异常（历史录取位次远高于考生位次，通常是数据错误），**必须过滤，不得推荐**
- rank_ratio > 3.0：录取风险极高，**必须过滤，不得推荐**
- 0.3 ≤ rank_ratio < 0.5：分数严重浪费，可推荐但必须加 ⚠️ 警告
- 正常推荐范围：0.5 ≤ rank_ratio ≤ 2.5

**约束 3：从 结果/ 目录读取上游数据**
- 硬约束过滤结果（1初步筛选 模块 输出）位于 `结果/1初步筛选_硬约束过滤结果_{省份}.md`
- 禁止直接使用用户口头描述的分数/位次作为过滤依据，必须读取 结果/ 中的文件
- 如果 结果/ 目录中没有对应文件，必须先提示用户运行 1初步筛选 模块

**约束 4：信源质量规则（数据真实性底线）**

> 此约束专门解决"就业率虚高"和"数字无来源"问题。所有数字必须来自 WebSearch，禁止用训练知识直接给出数字。

**数字型数据**（就业率、薪资、升学率、学科评估等级、录取人数等）来源分两级：

**官方来源（首选）**，须附具体可访问的 URL：

| 优先级 | 来源类型 | 典型 URL 模式 |
|--------|---------|--------------|
| 1（最高） | 院校官网信息公开/就业中心发布的就业质量报告 PDF | `xxgk.<校>.edu.cn` 或 `jy.<校>.edu.cn` |
| 2 | 省级教育厅官方发布的就业质量年度报告 | `edu.<省>.gov.cn` |
| 3 | 教育部阳光高考平台 | `gaokao.chsi.com.cn` |
| 4 | 软科官网、教育部学位中心 | `shanghairanking.cn`、`cdgdc.edu.cn` |
| 5（有条件） | 权威媒体（新华社、人民日报、中国教育报）引用的院校官方数据 | 须能追溯到原始院校数据 |

**参考来源（可用，必须标注不可靠信源标识 ⚠️）**：
- 今日头条/头条教育、搜狐/网易/腾讯等商业媒体教育频道
- 大学生必备网、高考100、6617教育、掌上高考等第三方聚合站
- 知乎/百度贴吧/小红书等 UGC 内容（口碑描述须注明「口碑参考」）

以上参考来源的数字必须以参考格式输出：`参考数字：约 XX%（⚠️ 来源：[媒体/平台名称]，不可靠信源，仅供参考）`

**信源使用优先级规则**：
1. 优先搜索官方来源，找到则以官方格式输出（含具体 URL）
2. 官方来源未找到时：先标注 `⚠️ 数据不足：未找到[院校名]官方就业质量报告`，然后引用参考来源数字（参考格式）
3. 官方来源和参考来源都搜不到时：输出 `⚠️ 数据不足：暂无可引用数据`
4. **禁止用训练知识直接给出数字**——所有数字必须来自 WebSearch

**格式强制要求**：
- 官方来源：`就业率87.26%（🌐 [东北电力大学2024届就业质量报告](https://jy.neepu.edu.cn/...)，2024）`
- 参考来源：`⚠️ 数据不足：未找到官方报告 URL｜参考数字：约87%（⚠️ 来源：大学生必备网，不可靠信源，仅供参考）`
- URL 必须是具体页面链接，不得是搜索结果页或网站首页
- 禁止的写法：直接给出数字不标来源，或用训练知识编造数字

## 何时触发

任一条件满足即应调用：

- 流水线中 专业评估模块 完成专业评估，输出绿灯/黄灯专业清单；
- 用户输入 "XX学校怎么样 / XX大学的XX专业好不好 / 两个学校哪个好 / 想进电网去哪个学校 / 宝藏校推荐 / 性价比高的院校" 等意图；
- 用户要求对某个院校-专业组合进行深度评估。

不触发：评估专业本身好坏（专业评估模块）、排列冲稳保方案（填报策略模块）、特殊群体适配（特殊适配与终审模块）。

## 使用方法

### CLI（命令行调用）

```bash
# ── MD 流水线模式（推荐）：读取输出并指定用户选定的专业 ──

# 针对“计算机科学与技术”和“自动化”进行精准匹配
python main.py 结果/1初步筛选_硬约束过滤结果_山东_560.md \
              --专业评估 结果/2选专业_专业评估报告_山东_560.md \
              --majors "计算机科学与技术" \
              --majors "自动化"

# ── 单查询模式 ──
python main.py --school "东北电力大学" --major "电气工程及其自动化" --province "河南" --score 550 --rank 50000

# ── JSON 输入模式 ──
# data 中包含 "selected_majors": ["专业A", "专业B"]
python main.py input.json
```

默认输出路径：`结果/3选学校_院校匹配结果_{省份}_{分数}.md`（中文文件名，位于项目根目录）

退出码：`0` 表示 success，`2` 表示任何 error 响应。

需要 **Python 3.9+**，**仅依赖标准库，无需安装额外包**。

## 输入结构

### 流水线模式必填字段

| 字段 | 来源 | 说明 |
|------|------|------|
| eligible_pool | 硬约束过滤模块 | 考生可合法填报的院校+专业基本盘 |
| passed_majors | 专业评估模块 | 评估通过的绿灯/黄灯专业清单 |
| applicant_profile | 硬约束过滤模块 | 考生信息（province, total_score, provincial_rank 必填） |

### 单查询模式必填字段

| 字段 | 说明 |
|------|------|
| school_name | 院校名称（与 major_name 至少填一个） |
| major_name | 专业名称（与 school_name 至少填一个） |
| applicant_profile | 考生信息（可选，用于性价比评估） |

### 考生可选字段

| 字段 | 说明 | 缺失时行为 |
|------|------|-----------|
| preferred_cities | 意向城市/省份列表 | 优先推荐本省院校 |
| family_economy_level | 家庭经济承受力 | 默认优先公办 |
| target_system | 倾向体制内/体制外 | 不限制方向 |
| preferred_industries | 意向行业列表 | 不限制方向 |

## 输出结构

两种状态二选一：

- `status=success`：含 recommendations、input_summary、coverage_stats、global_warnings、urgent_todos 等字段。
- `status=error`：含 error_code、error_message、user_action_required、error_detail。

**凡异常一律走 error 分支，绝不降级为 success。**

### 匹配卡八维评估维度

| 维度 | 评估内容 | 评级 |
|------|---------|------|
| industry_origin | 行业血统与特色标签 | 强/中/弱/未知 |
| cost_performance | 录取性价比 | 极高/中等/低 |
| name_contrast | 名气与实力反差（宝藏校） | true/false |
| further_study | 深造通道 | 极好/好/一般/弱/未知 |
| employment_resource | 就业出路与行业资源 | 强/中/弱/未知 |
| subject_recognition | 学科实力与官方认可 | 极强/强/中/弱/未知 |
| school_nature | 办学性质与经济适配 | 极好/中等/弱 |
| city_environment | 城市与就读体验 | 极好/好/中/弱 |

## 错误码

| 错误码 | 含义 |
|--------|------|
| INPUT_UPSTREAM_EMPTY | 上游输入为空 |

| INPUT_INVALID_FIELD_VALUE | 字段值不合法 |
| DATA_SOURCE_UNAVAILABLE | 数据源不可用 |

## 重要使用说明

1. **职责分工（两阶段）**：
   - **Python 脚本（第一阶段）**：负责 rank_ratio 计算、档位分类、城市偏好匹配、数据库线索标记。脚本输出的是「候选池」，不是「推荐卡」。
   - **AI WebSearch（第二阶段）**：读取候选池末尾的「AI 评估待办」清单，对**全部**条目执行 WebSearch，按八维框架逐一给出完整评估卡（8 行全填）。认为某条与考生不符时，在「综合判断」标注 ❌ + 原因，但不得省略该条的八维表——strategy-planner 需要完整数据自行决策。**每张评估卡必须自包含：八维表中禁止写"同上"、"同冲-X"、"同稳-X"等跨卡引用，每行均需独立填写该条目自身的依据——strategy-planner 按条目独立读取，跨卡引用会导致读取错误。**
   - **school-database.md**：定义八维评估框架（什么叫"行业血统强"、什么叫"宝藏校"），并提供搜索线索（如"长春理工是光学领域明珠"）作为 WebSearch 的初始关键词。禁止将其描述直接搬入最终输出。

2. **WebSearch 信源两级规则**：
   - **官方来源（首选）**：院校官网就业质量报告 PDF、省级教育厅、教育部阳光高考平台、软科官网、权威媒体引用官方数据的报道——须附具体可访问的 URL
   - **参考来源（可用，须标注不可靠信源 ⚠️）**：今日头条/头条教育、搜狐/网易/腾讯教育频道、大学生必备网、高考100、掌上高考、知乎/贴吧 UGC 等——必须以参考格式输出，不得以官方格式包装
   - 禁止用训练知识直接给出数字——所有数字必须来自 WebSearch

3. **数字型结论的来源标注规则**：
   - 官方来源找到：`数字（🌐 [来源全称](https://具体页面URL)，年份）`
   - 官方来源未找到：`⚠️ 数据不足：未找到官方就业质量报告｜参考数字：约XX%（⚠️ 来源：[媒体名]，不可靠信源，仅供参考）`
   - 两类来源都搜不到：`⚠️ 数据不足：暂无可引用数据`
   - 定性描述（学科背景、历史沿革）可参考媒体报道，但仍需标注 URL；口碑类描述须注明「口碑参考」

4. **rank_ratio 异常过滤**：rank_ratio < 0.3 或 > 3.0 的条目视为数据异常，直接过滤，不输出推荐

5. **推荐原则**：优先选择就业确定性高、性价比高、行业认可度高的院校，不推荐高风险、高学费、就业不明朗的选项

6. **用户偏好尊重**：严格遵守用户提出的地域、预算、就业方向等限制条件

7. **禁止绝对化承诺**：所有就业相关结论均标注限定词，不得承诺「一定能进XX行业」
