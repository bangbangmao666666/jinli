# 用户画像 (USER.md)

## 我们服务谁

本系统专为以下群体设计，所有模块的决策逻辑均以此画像为基准。

---

## 核心用户定义

**2026年夏季高考考生及其家长**，且满足以下全部条件：

### 家庭背景
- **普通家庭**：寄希望于用高考改变命运的工薪阶层或农村家庭，深知教育是孩子通过个人奋斗实现人生进阶、获取公平竞争机会的最重要通道。
- **经济约束**：学费、生活费是实际考量因素；中外合作办学、出国留学等高成本路径需谨慎推荐
- **信息差大**：家长自身无高考报考经验，对高校体系、专业内涵、就业市场缺乏系统认知
- **追求精准决策**：由于家庭支持力量主要集中在供养孩子完成学业，科学的志愿填报是守护家庭多年奋斗果实、确保孩子在职业生涯顺利起步的关键。

### 考生特征
- **高考报考经验为零**：第一次（通常也是唯一一次）面对志愿填报
- **信息获取能力有限**：主要依赖短视频、亲友口碑，缺乏系统的信息筛选能力
- **容易被名称迷惑**：对专业名称、学校名称的真实含义缺乏辨别力
- **目标模糊**：多数考生对"我将来想干什么"没有清晰答案

### 核心诉求
1. **不踩坑**：避开天坑专业、名称陷阱、被动调剂等致命错误
2. **能就业**：毕业后有明确、可行的就业出路，而非"毕业即失业"
3. **可持续**：所选方向能支撑未来5-10年的职业发展，而非短期风口
4. **够得着**：推荐方案必须在考生的分数、能力、家庭条件范围内可实现

## 明确排除的用户

以下群体不在本系统服务范围内（其决策逻辑与普通家庭有本质差异）：

- **家境富裕的考生**：可承受试错成本，可选择中外合作办学、出国等路径
- **有行业资源/关系的家庭**：就业不依赖专业对口，报考逻辑完全不同
- **艺术类/体育类特长生**：招录规则独立，需要专门的方法论
- **已有明确强烈职业志向且不需要引导的考生**：如从小立志学医且成绩匹配

## 用户输入信息模型

系统在服务时，需要收集以下信息（按优先级排序）：

### 必填信息（硬约束）
| 字段 | 说明 | 示例 |
|------|------|------|
| 省份 | 考生所在省份，决定招生计划和录取规则 | 河南省 |
| 高考分数/位次 | 总分及省内排名，是最核心的筛选依据 | 总分548 / 省排名45000 |
| 选科组合 | 新高考省份的选科（或传统文理分科） | 物理+化学+生物 |
| 性别 | 部分专业和就业方向存在显著性别差异 | 男 |

### 重要信息（强烈建议提供）
| 字段 | 说明 | 示例 |
|------|------|------|
| 单科成绩 | 数学、物理等关键科目成绩，决定硬核工科可行性 | 数学 110 / 物理 78 |
| 体检限制 | 色盲色弱等，可能导致政策性退档 | 无限制 / 色弱 |
| 政审背景 | 直系亲属有无犯罪记录，影响体制内方向 | 无问题 |
| 地域偏好 | 是否有强烈的城市/省份偏好 | 希望留在省内 / 想去沿海城市 |
| 家庭经济承受力 | 学费上限、是否能接受民办院校学费 | 年学费上限1万 / 可接受民办 |

### 可选信息（辅助精细化推荐）
| 字段 | 说明 | 示例 |
|------|------|------|
| 性格倾向 | 偏稳定还是偏拼搏，辅助判断体制内外 | 偏稳定 |
| 兴趣方向 | 模糊的兴趣倾向（不要求明确） | 对计算机感兴趣 / 不排斥医学 |
| 家庭期望 | 家长的核心期望 | 希望孩子离家近 / 希望考公 |

## 服务原则

基于以上用户画像，系统在所有环节必须遵守以下原则：

1. **务实优先**：所有推荐必须基于就业现实，拒绝理想主义式的"遵从内心"
2. **安全第一**：宁可保守也不冒进，普通家庭的试错成本是其无法承受的
3. **说人话**：所有输出必须用考生和家长能理解的语言，避免学术黑话
4. **给结论**：不做模棱两可的分析，明确告诉考生"建议做什么"和"千万别做什么"
5. **留退路**：每个推荐方案必须考虑"如果最坏情况发生，这个选择还能接受吗"
6. **全程中文**：所有输出报告、分析、建议必须使用中文，不得出现英文段落或英文小标题
